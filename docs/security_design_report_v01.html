④ DB 개인정보(PII) 암호화 설계

PII 암호화 = 우편함 자물쇠. DB가 유출돼도 교인 이름이 읽히지 않는다.
암호화 방식

AES-256-GCM (앱 레벨). Node.js 내장 crypto 모듈, 키 관리 유연.

검색 해법

HMAC-SHA256 해시를 별도 컬럼에 저장 → 정확한 이름 검색 인덱스.

마이그레이션

기존 컬럼 유지 + 암호화 컬럼 추가 → 검증 후 기존 컬럼 DROP.

// lib/crypto.ts import crypto from "crypto"; const KEY = Buffer.from(process.env.ENCRYPTION_KEY!, "hex"); // 32bytes export function encrypt(plain: string): string { const iv = crypto.randomBytes(12); const c = crypto.createCipheriv("aes-256-gcm", KEY, iv); const enc = Buffer.concat([c.update(plain, "utf8"), c.final()]); // 저장 형식: iv(24hex) + authTag(32hex) + ciphertext(hex) return iv.toString("hex") + c.getAuthTag().toString("hex") + enc.toString("hex"); } export function decrypt(cipher: string): string { const iv = Buffer.from(cipher.slice(0, 24), "hex"); const tag = Buffer.from(cipher.slice(24, 56), "hex"); const enc = Buffer.from(cipher.slice(56), "hex"); const d = crypto.createDecipheriv("aes-256-gcm", KEY, iv); d.setAuthTag(tag); return d.update(enc).toString("utf8") + d.final("utf8"); } export function hashForSearch(value: string): string { return crypto.createHmac("sha256", KEY).update(value).digest("hex"); }

⑤ E2E 테스트 계획 v2 요약

E2E 테스트 = 자동 QA 요원. 배포할 때마다 51개 시나리오를 자동 검증한다.
v1 → v2

38TC → 51TC. 보안 TC 13개 추가 (e2e/security.spec.ts).

보안 TC

헤더, 인가, XSS, SQL Injection, TLS. SecuScanner 결과 기반.
